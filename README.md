# brain

A personal memory vault for Claude Code. Sessions stop resetting. Knowledge compounds.

Built on the [3-layer memory architecture](https://x.com/nyk_builderz/article/2030904887186514336):

- **Layer 1 — Session memory**: `CLAUDE.md` + auto-memory directory tell Claude who you are and what you've decided
- **Layer 2 — Knowledge graph**: Obsidian vault + MCP bridge lets Claude query your notes at runtime
- **Layer 3 — Ingestion pipeline**: `brain-ingest` converts YouTube/audio/transcripts into structured vault notes

---

## Requirements

[Claude Code](https://claude.ai/code) must be installed before running setup.

**macOS** — Node.js and ffmpeg are installed automatically if Homebrew is present. Python 3.11+ must be installed first:

```bash
brew install python@3.12
```

**Linux (Debian/Ubuntu)**:

```bash
sudo apt-get update
sudo apt-get install -y python3 python3-venv nodejs ffmpeg
```

**Linux (Fedora/RHEL)**:

```bash
sudo dnf install -y python3 nodejs ffmpeg
```

**Linux (Arch)**:

```bash
sudo pacman -S python nodejs ffmpeg
```

[Obsidian](https://obsidian.md) is optional — the vault is plain markdown files and works without it.

## Setup

```bash
git clone https://github.com/<you>/<your-repo-name>
cd <your-repo-name>
bash setup.sh   # or: zsh setup.sh
```

That's it. `setup.sh` does everything:

- Installs Node.js and ffmpeg (macOS with Homebrew only; Linux: prompts with the right package manager command)
- Creates the Python venv and installs `brain-ingest` / `brain-persist`
- Prompts for your Anthropic API key and saves it to `.env`
- Installs `brain-ingest` and `brain-persist` wrappers to `~/.local/bin/`
- Registers the `brain-obsidian` MCP server with Claude Code at user scope (all projects)
- Installs the `brain-persist` Stop hook into `~/.claude/settings.json`
- On every session end, `brain-persist` extracts learnings and **pushes the vault to your Git remote**

After setup, open the `brain/` folder in Obsidian (**File → Open folder as vault**) and restart Claude Code.

To uninstall:

```bash
./uninstall.sh
```

This removes all tooling (MCP server, hooks, wrappers, venv). Your vault notes are never deleted.

## Usage

```bash
# Ingest a YouTube video (downloads + transcribes locally, extracts with Claude)
brain-ingest "https://youtube.com/watch?v=..." --apply

# Ingest a local audio or video file
brain-ingest /path/to/recording.mp4 --apply

# Ingest a text or markdown file as a transcript
brain-ingest /path/to/notes.md --transcript --title "Team Retro" --apply

# Dry-run — preview the extracted note without writing it
brain-ingest "https://youtube.com/watch?v=..."
```

Whisper transcription runs fully locally — no audio leaves your machine.

## How it works

### Vault structure

```
brain/
├── 00-home/               # Maps of content and active attention
├── atlas/                 # Structural indexes (projects, research, vault arch)
├── inbox/                 # Unprocessed captures — review and promote
│   └── queue-generated/   # brain-ingest drop zone
├── knowledge/
│   ├── graph/
│   │   ├── agent-daily/   # Agent observations
│   │   ├── research/      # Curated research notes
│   │   └── repo-research/ # Codebase-specific knowledge
│   └── memory/            # Persistent cross-session memory
├── sessions/              # Session learning notes (auto-generated on Stop)
└── voice-notes/           # Transcribed voice captures
```

### Note conventions

- **Prose-as-title**: name notes as claims — `memory graphs beat giant memory files.md`, not `memory-systems.md`
- **Wiki-links as prose**: `we learned that [[memory graphs beat giant memory files]]`
- **Atomic notes**: one idea per note, compose complexity through links

### Claude Code integration

| Integration | Mechanism | Effect |
|---|---|---|
| Vault context | `CLAUDE.md` loaded at session start | Claude knows the vault exists and its conventions |
| Read/write vault | `brain-obsidian` MCP server (user scope) | Claude can search and write notes in any project |
| Persist learnings | `brain-persist` Stop hook (global) | Key decisions extracted to `sessions/` and pushed to Git on every session end |

The vault path is stored in `~/.config/brain/vault`. Wrapper scripts in `~/.local/bin/` read it at runtime — no hardcoded paths in any config file.
